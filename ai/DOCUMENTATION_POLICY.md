# Documentation Policy

Every feature must be documented.

Documentation is mandatory.

---

# PROJECT_PLAN.md

Purpose:

Maintain project roadmap.

Contains:

- Phases
- Milestones
- Planned features
- Dependencies

Update when:

- New feature added
- Scope changes
- Roadmap changes

---

# FUNCTION_REGISTRY.md

Purpose:

Maintain functionality history.

Every feature must contain:

- Purpose
- Files
- Version
- Changes

Format:

## Feature Name

### Version 1.0

Purpose

Files

Behavior

### Version 1.1

Changes

Previous Behavior

Current Behavior

---

# DEV_LOG.md

Purpose:

Track implementation history.

Every update must include:

Date

Feature

Reason

Files Modified

Risk Level

Result

---

# KNOWLEDGE_BASE.md

Purpose:

Store technical project knowledge.

Contains:

- Architecture
- Services
- Data flow
- APIs
- Storage
- Background systems

Update only when architecture changes.

---

# DECISIONS.md

Purpose:

Store architectural decisions.

Every decision must include:

- Context
- Decision
- Reason
- Alternatives considered
- Consequences

This file prevents repeated debates and inconsistent implementations.
